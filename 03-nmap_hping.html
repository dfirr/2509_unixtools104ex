3  パケットの操作（nmap, hping）

公開

2026年1月20日

更新日

2026年2月25日

3.1 目的

この章の目的は、次の通りです。

  • nmapによる自動化されたスキャンの基本が理解できるようになること。
  • nmapのscript機能を使って、対象ホストの追加情報を得られるようになること。
  • hping3を使って、より低いレベルでパケットを操作できるようになること。

3.2 この章の注意事項

この章では、インターネット上のホストに対してスキャンしません。演習はすべてDockerの閉域ネットワーク内で実施します。

組織内ネットワークやインターネットに対する無断スキャンは、セキュリティ事故や規約違反につながるおそれがあります。必ず、講義用に準備された環境のみを対象にしてください。

3.3 事前準備（git, Docker）

この章は、WSL上のUbuntuを前提に進めます。まず、gitとDocker関連ツールをインストールします。

sudo apt update
sudo apt install -y git docker.io docker-compose-v2

Dockerをsudoなしで使えるように、ユーザーをdockerグループに追加します。

sudo usermod -aG docker $USER

この変更は、ログインし直すと反映されます。WSLターミナルを閉じて開き直すか、次のコマンドで現在のシェルに反映してください。

newgrp docker

以下で動作確認します。

git --version
docker --version
docker compose version

3.4 演習ファイルの取得

GitHubから演習ファイルを取得します。

mkdir -p ~/cases
cd ~/cases
git clone https://github.com/dfirr/2509_unixtools104ex.git
cd 2509_unixtools104ex/ch3-lab

3.5 演習ラボの起動

次のコマンドで演習環境を起動します。

docker compose up -d --build

起動するコンテナは以下の3つです。

  • ch3-scanner: nmap, hping3, tcpdump, nc, curlを実行する側
  • ch3-target-web: 80/tcpを待ち受けるWebサーバー側
  • ch3-target-closed: 待ち受けポートを持たない側

正常に起動できたかを確認します。

docker compose ps

3.6 scannerで使えるコマンド確認

scannerコンテナに入り、この章で使うコマンドを確認します。

docker exec -it ch3-scanner bash
which nmap hping3 tcpdump nc curl

確認できたら、以後のコマンドはscannerコンテナ内で実行します。

3.7 nmapの基本

nmapは、ネットワーク探索とポートスキャンを行うためのツールです。基本的な書式は次の通りです。

nmap [オプション] 対象

まずはtarget-webに対して、最小限のスキャンを実行します。

nmap target-web

出力では、次の列に注目してください。

  • PORT: ポート番号とプロトコル
  • STATE: ポート状態（open/closed/filtered）
  • SERVICE: 推定されたサービス名

3.8 ホスト発見（sweeping）

-snオプションを使うと、ポートスキャンを行わず、ホストが応答するかどうかを確認します。

nmap -sn target-web target-closed

同一セグメント内をまとめて探索するときには、CIDR表記を使います。

nmap -sn 172.21.0.0/24
ノート

ここでいうsweepingは、アドレス範囲を走査して「どのホストが生きているか」を確認する手法です。実運用環境では必ず許可を得た範囲だけで実施してください。

Problem問題

nmap -sn 172.21.0.0/24の結果から、応答したホスト数を確認してください。target-webとtarget-closedは見つかるでしょうか？

3.9 ポートスキャン基礎

-pで対象ポートを指定します。

nmap -p 80 target-web

-sTはTCP connect scanです。

nmap -sT -p 80 target-web

-sSはSYN scanです（通常は管理者権限が必要）。

nmap -sS -p 80 target-web

-Pnを使うと、事前のホスト発見を省略してスキャンします。

nmap -Pn -p 80 target-web
ノート

-sTはOSの通常ソケットAPIを使って接続を試みます。一方、-sSはSYNパケットを直接使った半開接続（half-open）の手法で、より低い層の制御を行います。

3.10 状態推定（open/closed/filtered）

nmapの出力では、ポート状態としてopen, closed, filteredが表示されます。target-webとtarget-closedを比較し、結果の違いを確認してください。

nmap -p 80,443 target-web target-closed

大まかな意味は次の通りです。

  • open: 対象ポートで待受しているプロセスがある
  • closed: 対象ホストには到達できるが、待受はない
  • filtered: フィルタリング等により状態判定が難しい

3.11 ACKスキャン（-sA）

-sAはACKパケットを使って、フィルタリングの有無を推定する手法です。

nmap -sA -p 80 target-web

結果にunfilteredやfilteredが出たとき、それが何を意味するかを考えましょう。

nmap -sA -p 80 target-closed
ヒント観察のポイント

-sSと-sAとでは、得られる情報の種類が異なります。-sSはポート開閉の推定に、-sAはフィルタの有無の推定に向いています。

3.12 バージョン検出・OS推定

-sVでサービスバージョン検出を行います。

nmap -sV -p 80 target-web

-OはOS推定です。コンテナ環境では精度に限界があるため、推定結果が不正確になる場合があります。

nmap -O target-web

3.13 NSE入門（–script）

nmapにはNSE（Nmap Scripting Engine）があり、用途別のスクリプトを実行できます。

nmap --script-help default

--scriptでカテゴリや個別スクリプトを指定します。

nmap -p 80 --script default target-web

HTTP関連スクリプトの例を確認してみます。

ls /usr/share/nmap/scripts/http-* | head

3.14 HTTP系スクリプト演習

HTTP系スクリプトを使って、Webサーバーの情報を収集します。

nmap -p 80 --script http-title,http-headers,http-server-header target-web

さらにhttp-enumで、公開されているパスを探索します。

nmap -p 80 --script http-enum target-web
ノート

http-enumは、よく使われるパス候補に対してHTTPリクエストを送り、応答コードなどから存在を推定します。

3.15 演習：admin.htmlを見つける

http-enumで検出された候補を、curlで実際に確認します。

curl -i http://target-web/admin.html
Problem問題

/admin.htmlは、target-webのトップページからリンクされているでしょうか。また、攻撃者がこのページを発見できる理由を説明してください。

ヒントヒント

curl -i http://target-web/robots.txt を実行してみてください。

ヒント解答

リンクが無くても、推測しやすいURLや列挙スクリプトの辞書探索により発見される可能性があります。robots.txtの記述も手がかりになります。

3.16 tcpdump併用でスキャンを裏付ける

nmapの結果を「文字列」だけで終わらせず、パケットとして確認します。1つのターミナルでtcpdump、もう1つでnmapを実行してください。

tcpdump -n -i eth0 'host target-web and tcp port 80'
nmap -sS -p 80 target-web
ノート

-sSでは、SYN -> SYN/ACK（またはRST）という往復に注目してください。Ch.2で学んだ内容と接続すると理解しやすくなります。

3.17 hping3入門

hping3は、任意のフラグを付けたTCPパケットなどを手動で送信できるツールです。nmapより低レベルの実験ができます。

まずはSYNパケットを送信します。

hping3 -S -p 80 -c 3 target-web

-SはSYNフラグ、-pは宛先ポート、-cは送信回数です。

3.18 hping3演習（SYN/ACK/フラグ操作）

SYN以外のフラグでも挙動を確認します。

hping3 -A -p 80 -c 3 target-web
hping3 -F -p 80 -c 3 target-web
Problem問題

-Sと-Aと-Fとで、応答の傾向はどのように変わるでしょうか。tcpdumpと併用して観察し、理由を説明してください。

ヒント観察の補助

別ターミナルで次を実行し、フラグの違いを確認してください。

tcpdump -n -i eth0 'host target-web and tcp'

3.19 追加課題

次の2つの立場で、同じ対象を調査してください。

  • 立場A: nmap中心（自動化重視）
  • 立場B: hping3中心（手動制御重視）

それぞれについて、次の観点で比較しなさい。

  • 実行の速さ
  • 得られる情報の量
  • 手順の再現性
  • 誤操作リスク

3.20 まとめ

この章では、次の3点を学習しました。

  • nmapで対象のホスト・ポート・サービスを効率的に調査する方法
  • NSE（--script）で追加情報を得る方法
  • hping3でTCPフラグを操作し、応答を観察する方法
